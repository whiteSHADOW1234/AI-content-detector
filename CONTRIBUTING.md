# Contributing to AI-content-detector

Thank you for your interest in contributing to the AI-content-detector project! We welcome contributions from everyone. Please read the following guidelines to help us maintain a collaborative and productive environment.

## How to Contribute

1. **Fork the Repository**
   - Click the "Fork" button at the top right of this repository to create your own copy.

2. **Clone Your Fork**
   - Clone your forked repository to your local machine:
     ```bash
     git clone https://github.com/<your-username>/AI-content-detector.git
     ```

3. **Create a Branch**
   - Create a new branch for your feature or bugfix:
     ```bash
     git checkout -b feature/your-feature-name
     ```

4. **Install Dependencies**
   - Install the required Python packages:
     ```bash
     pip install -r requirements.txt
     ```

5. **Make Changes**
   - Implement your changes. Please follow the existing code style and add comments where necessary.

6. **Test Your Changes**
   - Ensure your changes work as expected. If possible, add tests or screenshots to demonstrate your changes.

7. **Commit and Push**
   - Commit your changes with a clear message:
     ```bash
     git commit -m "Add feature: your feature description"
     git push origin feature/your-feature-name
     ```

8. **Create a Pull Request**
   - Go to the original repository and open a Pull Request (PR) from your branch. Provide a clear description of your changes.

## Commit Message Guidelines
- Please use [Conventional Commits](https://www.conventionalcommits.org/) for your commit messages. This helps keep the project history clear and automates changelog generation.
- Example commit message formats:
   - `feat: add new analysis feature`
   - `fix: resolve crash on startup`
   - `docs: update README`
   - `refactor: improve code structure`
- See the Conventional Commits website for more details and types.
